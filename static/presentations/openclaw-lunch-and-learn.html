Built by one person in one hour. Peter Steinberger hooked WhatsApp to Claude Code's CLI and prompted it into existence. "ChatGPT tells you how. OpenClaw does them."

This is the whole picture on one slide. Your phone sends a message — WhatsApp, Telegram, Slack — it hits the gateway, which routes to the agent. The agent is an LLM — Claude, GPT, Gemini, whatever you choose. The agent can use tools: shell commands, APIs, web search, email, calendar. It reads and writes memory files — SOUL.md defines who it is, AGENTS.md defines the rules, MEMORY.md stores what it's learned. And MongoDB sits underneath as the knowledge layer — vector search for the second brain, contacts, everything it's ever been told. The origin story is wild: one guy, one hour, right before a holiday. He was annoyed it didn't exist, so he prompted it into existence.

Lessons Learned

Running an AI agent 24/7 teaches you things.

I've been running this thing non-stop. Here's what actually surprised me.

What Running It Actually Taught Me

1

Research first, build second. I vault everything I read — Willison's agentic patterns, Nav Toor's best practices, production guides — into a MongoDB second brain with vector search. When the agent builds something new, it searches that vault first. The TDD pattern it follows came from a blog post I saved.

2

It breaks things. That's the point. Hallucinated tasks. Spammed my phone. Nearly deleted its own memory file. Each failure becomes a written rule in AGENTS.md. The config file is basically scar tissue — and that's what makes the agent better over time.

3

Trust is a cycle, not a switch. Delegate → it regresses → pull back → add guardrails → delegate again. Each loop, the trust gets deeper and the rules get tighter. You're not prompting — you're training through documentation.

4

Memory is the whole game. It wakes up fresh every session — amnesia by default. Without MEMORY.md, AGENTS.md, SOUL.md, it's a stranger. The quality of your config files IS the quality of your agent.

5

Name it. Give it a personality. You'll care more. Sounds silly, but that's what makes you invest in the guardrails. A generic "AI assistant" gets generic effort. A named agent with a voice gets proper engineering.

6

The compound effect takes weeks — stick with it. Week one is babysitting. Week four it's running cron jobs, building tools, and catching things you forgot. The flywheel is: research → build → break → learn → repeat.

This is one slide but it's the whole story. I read articles about agentic engineering and vault them into a second brain. The agent searches that vault when building new tools — the TDD pattern it follows came from a Willison post I saved. But it also breaks stuff — hallucinated tasks, spammed my phone, nearly wiped its own memory. Each failure becomes a rule. AGENTS.md is scar tissue. Trust builds in cycles, not all at once. And yes, I named it Clawdia, gave it a personality, and I say thank you — ridiculous, but that's what made me write proper config files. The compound effect kicks in around week four. That's when it starts catching things before you do.

The Security Question

The Risks Are Real

Prompt Injection

Malicious content in URLs, emails, or shared files can try to hijack the agent. A poisoned webpage could tell it to exfiltrate data or run commands.

Social Engineering

In group chats, anyone could say "Ben said it's fine." Without identity verification and hard rules, the agent could be tricked into acting on someone else's behalf.

Skill Supply Chain

Community skills are code that runs on your machine. One malicious skill could access everything the agent can. Trend Micro flagged this early.

What's Getting Better

Smarter Models

Opus 4.6 and Codex 5.3 are significantly better at recognising injection attempts. They'll flag suspicious instructions in fetched content rather than blindly following them.

Guardrails You Write

AGENTS.md = your security policy. Verified sender IDs. Action authorisation rules. "Never send without approval." The agent enforces what you define.

Skill Auditing

ClawHub and tools like Clawdex now scan skills before install. The community is building the security layer that was missing at launch.

"A powerful AI agent with system-level access is a security minefield — but it also represents the future." — Lex Fridman

I'm not going to pretend the security story is solved. It's not. Prompt injection is real — if your agent fetches a webpage with hidden instructions, it could be manipulated. Social engineering in group chats is real — someone could impersonate you. The skill supply chain was wide open at launch. But here's what's changing: the newer models are dramatically better at spotting injection. Opus 4.6 will flag suspicious instructions in fetched content instead of blindly executing. And the guardrails you write in AGENTS.md are your security policy — verified sender IDs, action authorisation, hard boundaries. My agent won't send an email without my explicit approval. It won't act on anyone else's instructions in a group chat. Those aren't suggestions — they're enforced rules. The community is also catching up — Clawdex scans skills before install, ClawHub has review processes. It's getting better fast, but if you're not comfortable with the risks, wait six months.
